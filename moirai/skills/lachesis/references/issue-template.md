# Sub-Issue Template

This is the same template used by the clotho (plan) phase. Provided here for reference when implementers need to understand the sub-issue structure.

```markdown
# <Sub-Issue Title>

## Type

[HITL or AFK]

## Parent PRD

<linear-issue-id>

## What to Build

[End-to-end behavior description for this vertical slice.]

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Blocked By

- <issue-id> — <brief reason>

Or "None — can start immediately" if no blockers.

## User Stories Addressed

- User story 1
- User story 3
```
